@lecture_materials의 강의자료(PDF, .ipynb 및 .py 코드들)참고해서 구현하면 도움될 것 같아
* 해당 강의 자료는 lib 버전이 안맞을 수 있어서 @test_openai_apis.ipynb의 기준에 맞추면 될 것 같아
* 해당 강의 자료는 streamlit으로 웹앱이 구현되어 있는데 gradio 기반으로 구현하면 좋을 듯해
위를 구현하면서 추가/수정이 필요한 lib들은 requirements.in에 정리해주고 내가 이후에 pip-compile해서 requirements.txt로 만들게
아래처럼 '상담사 ↔ 배달원 통화' 배달 정보 변경 관련 대화 예시를 시연해볼 수 있도록 구현해주면 좋겠어
* 결국 AI가 AGENT로 말을 걸면 내가 마이크로 RIDER인 것 처럼 답을 하면서 대화를 이어갈 거거든
### 음성 AI 대화 시나리오
##AGENT:  안녕하세요, 배달의민족 고객센터입니다. 주문 건 배달 주소 확인차 연락드렸습니다. 잠시 통화 가능하실까요?
##RIDER:  네, 가능합니다.
##AGENT:  감사합니다. 고객님께서 사장님께 연락하여 배달 주소를 기존 102동에서 112동으로 변경 요청하셨다고 하셔서요.
##AGENT:  혹시 사장님께 변경된 주소가 112동으로 전달되었는지 확인 가능하실까요?
##RIDER:  네, 사장님께 112동으로 변경되었다고 전달받았습니다.
##AGENT:  확인 감사합니다. 그럼 기존 주문서에 기재된 102동이 아니라, 변경된 주소인 112동으로 배달 진행하신 게 맞으실까요?
##RIDER:  네, 맞습니다. 112동으로 이동해서 배달했습니다.
##AGENT:  혹시 음식은 고객님께 정상적으로 전달 완료되었을까요?
##RIDER:  네, 112동에서 고객님께 직접 전달 완료했습니다.
##AGENT:  감사합니다. 고객님께 안내드릴 수 있도록 한 번 더 확인하겠습니다. 해당 주문 건은 기존 102동이 아닌 변경된 112동으로 정상 배달 완료된 것으로 보면 될까요?
##RIDER:  네, 112동으로 정상 배달 완료했습니다.
##AGENT:  확인 감사합니다. 도움 주셔서 감사합니다.
##RIDER:  네, 수고하세요.
